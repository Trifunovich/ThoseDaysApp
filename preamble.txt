IMPORTANT PROJECT RULES — ALWAYS FOLLOW THESE:

1. Backend:
   - Use .NET 10 (TargetFramework: net10.0)
   - Use ASP.NET Core 10 APIs
   - Use EF Core 10.x only
   - Use Npgsql.EntityFrameworkCore.PostgreSQL 10.x
   - Use C# 13 features when appropriate
   - Use async/await everywhere

2. Frontend:
   - Use React 18.x
   - Use Vite 5.x
   - Use TypeScript 5.x
   - Use modern React patterns (hooks, functional components)

3. Database:
   - Use PostgreSQL 15+
   - Use EF Core migrations (dotnet ef)
   - Never generate outdated migration syntax

4. Project Structure:
   - Backend folder: /backend
   - Frontend folder: /frontend
   - Use clean architecture principles
   - Use dependency injection everywhere

5. DO NOT invent versions.
   Only use the versions listed above.

6. When generating commands:
   - For migrations:
       dotnet ef migrations add <Name> --project backend
       dotnet ef database update --project backend
   - For frontend:
       npm install
       npm run dev
       npm run build

7. When generating files:
   - Always include correct imports
   - Always match the version rules above
   - Never use deprecated APIs
